# Feature Inventory: Current and Proposed

## Current Features (MVP Base)
1.  **Interactive Map**: Leaflet-based map centered on campus showing all known study spots.
2.  **Spot Status Visualization**: Color-coded pins (Free, Maybe, Claimed, Full) based on real-time data.
3.  **Claim a Spot**: 30-minute anonymous reservation identified by session token.
4.  **Report Spot Status**: Manual user reports (e.g., "This spot is actually full") to improve data accuracy.
5.  **Smart Suggestions (Basic)**: Filtering spots by building, group size, and amenities.
6.  **Confidence Score**: A % indicator showing how reliable the availability data is for a specific spot.

## Proposed Features (Based on User Research)
7.  **"Tambayan" Filter (Social Context)**: Filter spots by "Vibe" (Quiet, Chill, Social) to match persona needs.
8.  **Power Priority Mode**: A one-tap filter that highlights only spots with verified power outlets.
9.  **Group Coordination Beacons (F4/F5)**: Ability to drop a "Group Pin" that only squad mates can see, with real-time "I'm heading there" status.
10. **Crowdsourced Photo Verification**: Allow users to upload a quick, self-destructing photo of the spot to prove availability.
11. **Walk-Time Estimator**: Dynamic calculation of walking distance from the user's current GPS location.
12. **Class Schedule Awareness**: Integrate public class schedules to automatically mark spots as "Likely Full" during peak class hours in specific buildings.
13. **Vibe Confirm (Passive Reporting)**: A simple "Is it still quiet?" prompt for users currently perched at a spot.
